# Handoff: Accuracy Grind (2026-05-07)

## Goal
Push damage calculation accuracy from 36% exact match to 100% against real Pokemon Showdown replays.

## Current State
- 192,000 games/sec, 120 tests passing, 0 type-chart failures
- 1330 species, 950 moves, 310 abilities, 50+ ability effects, 30+ item effects
- Differential testing: 36% exact, 86% close (±20%), 14% direction-only
- The "direction only" cases are caused by missing modifiers in BOTH the verification test AND the engine

## Task (priority order)

### 1. Enhance test harness to extract RNG outcomes from replays
PS protocol reveals actual outcomes. Extract:
- `|-crit|` before `|-damage|` → know if crit happened
- Exact damage number → reverse-engineer the roll used (observed ÷ base = roll%)
- No `|-damage|` after `|move|` → miss

Feed these as oracle inputs to our damage calc. This gives STRICT tests: given exact roll + crit, does our formula produce the exact observed damage?

### 2. Fix engine + verification for common missing modifiers
Top causes of non-exact matches:
- Knock Off: 97.5 BP when target has item (not 65)
- Acrobatics: 110 BP when user has no item
- Facade: 140 BP when user is burned/poisoned/paralyzed
- Terrain boosts: Electric/Grassy/Psychic = 1.3x for matching type (grounded only)
- Abilities: Adaptability (2x STAB), Tinted Lens (2x resisted), Download, Sword of Ruin, Tablets of Ruin
- Items: Expert Belt (1.2x on SE), type-boosting items (Mystic Water = 1.2x)
- Multi-hit moves: total = per-hit × hits

### 3. Scale up fixtures
Target 200+ replays. Run: `cd tools && python3 fetch-fixtures.py --count 200 --output ../tests/fixtures/replay_events/`

## Workflow
1. `cargo test -p pkmn-engine damage_matches -- --nocapture` — see stats
2. Pick most common "Direction only" cause
3. Fix in both verification AND engine
4. Run tests, confirm improvement
5. Commit per AGENTS.md format

## Architecture
- Engine: `crates/pkmn-engine/`, data: `crates/pkmn-core/`
- All data is compile-time const (generated by `tools/gen-data.py`)
- Verification tracks state from protocol events, calculates expected damage
- Fixtures: `tests/fixtures/replay_events/*.json` (event-based)
